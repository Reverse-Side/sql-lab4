## Опис предметної області
Розроблена інформаційна система призначена для зберігання, обробки та керування даними через веб-інтерфейс у вигляді REST API. Система підтримує багатокористувацький доступ, авторизацію, валідацію введених даних та безпечну взаємодію з базою даних.  

Система може використовуватися як серверна частина для різноманітних інформаційних систем: облікових, адміністративних, навчальних або корпоративних. Завдяки модульній архітектурі вона легко масштабується та дозволяє додавати нові сутності без змін основної логіки.

## Концептуальна модель
Основні сутності системи:
- Користувач (User)
- Предметні сутності модулів (Entities)
- Сервісні сутності (аудит, токени доступу)

Основні атрибути користувача: ідентифікатор, ім’я користувача, електронна пошта, пароль, дата реєстрації.  

Зв’язки між сутностями:
- Один користувач може створювати та змінювати багато записів (зв’язок 1:M).
- Предметні сутності можуть бути пов’язані між собою через зовнішні ключі.
- Усі зв’язки реалізовані на рівні ORM та фізично у PostgreSQL.

## Логічна схема
Логічна схема бази даних реалізована за допомогою ORM **SQLAlchemy**. Кожна таблиця описана у вигляді Python-класу з відповідними полями та типами даних.  

Використовуються такі елементи:
- первинні ключі (Primary Key);
- зовнішні ключі (Foreign Key);
- унікальні обмеження (UNIQUE);
- перевірки коректності даних.

База даних нормалізована до **третьої нормальної форми (3НФ)**. Усі дані атомарні, відсутні часткові та транзитивні залежності.


## Реалізація в PostgreSQL
Для зберігання даних використовується **PostgreSQL**. Фізична структура бази даних створюється автоматично на основі **ORM-моделей SQLAlchemy**.  

У моделях визначено:
- структуру таблиць;
- типи полів;
- зв’язки між таблицями;
- обмеження цілісності.

Під час запуску застосунку SQLAlchemy автоматично створює таблиці у базі даних PostgreSQL. Безпосереднє написання SQL-скриптів для створення таблиць не використовується.


## Налаштування Docker
Для розгортання застосунку використовується **Docker** та **Docker Compose**. Конфігурація міститься у файлі `docker-compose.yml`, де описано два основних сервіси:
- контейнер з PostgreSQL;
- контейнер із FastAPI-застосунком.

Для початкової ініціалізації бази даних використовується скрипт `init_db.sh`.  

Запуск проєкту здійснюється командою:
---bash
docker compose up --build


## SQL-запити
У даному проєкті ручні SQL-запити не використовуються, оскільки робота з базою даних здійснюється через ORM SQLAlchemy. Усі операції з PostgreSQL виконуються за допомогою ORM-методів, які автоматично транслюються у відповідні SQL-команди.Використання ORM забезпечує безпеку, зручність розробки та захист від SQL-ін’єкцій.
